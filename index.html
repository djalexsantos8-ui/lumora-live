Instalar VoiceMeeter Banana

Gratuito

Mixer de áudio virtual que permite rotear o mesmo som para dois fones Bluetooth simultaneamente.

VoiceMeeter Banana é um aplicativo de mixagem de áudio virtual gratuito (donationware — funcional 100%, sem limitações, sem marca d'água). Ele cria dispositivos de áudio virtuais no Windows que você pode rotear para múltiplas saídas físicas.

No nosso caso: A1 = Fone BT 1, A2 = Fone BT 2. Qualquer áudio que entrar pelo Virtual Input será enviado simultaneamente para os dois fones. Essencial para o monitoramento dos apresentadores sem compartilhamento de fones.

Instalar NDI Tools

Gratuito

Permite que o computador do PowerPoint transmita a tela via rede local para o OBS, sem cabos.

Instalar em DOIS computadores: o PC principal (que roda o OBS) e o PC do PowerPoint.

  • No PC do PPT: abrir NDI Scan Converter após instalação — ele começa a transmitir a tela automaticamente via rede local.
  • No PC principal: o plugin OBS-NDI (próxima tarefa) vai detectar essa fonte.
  • Ambos os computadores precisam estar na MESMA rede Wi-Fi ou cabo. Idealmente cabo para estabilidade.

Configurar Sony FX30 para modo USB Streaming

Qui 7/5

Ativar o modo que faz a câmera aparecer como webcam no Windows.

  • No menu da FX30: Network → USB → USB Connection Mode → PC Remote
  • Depois: Network → PC Remote Settings → Still Img Save Dest → Memory Card Only (para não travar)
  • Conectar via cabo USB-C → USB-A ou USB-C → USB-C diretamente na porta TRASEIRA do gabinete (não hub, não frente)
  • A câmera vai aparecer no Windows como Sony Camera no Gerenciador de Dispositivos
  • Resolução recomendada: 1080p / 30fps no streaming USB (suficiente para a live, menos carga no barramento USB)

Configurar Sony A7S III para modo USB Streaming

Qui 7/5

Mesmo processo da FX30, mas o menu é levemente diferente.

  • Menu → Setup (maleta) → USB → USB Connection Mode → PC Remote
  • Conectar em outra porta USB do gabinete — idealmente em controlador USB diferente do FX30 (portas frontais vs traseiras geralmente usam controladores diferentes num i9)
  • Verificar no Gerenciador de Dispositivos que AMBAS aparecem como dispositivos de câmera separados

Configurar Zoom H6 como interface de áudio padrão

Qui 7/5

Definir o H6 como entrada e saída de áudio do Windows.

  • Conectar o H6 via USB. Ele aparece como Zoom H6 no Windows — sem driver adicional necessário.
  • Botão direito no ícone de som da barra de tarefas → Sound Settings
  • Output: pode ficar na saída da placa-mãe OU no H6 (depende de onde os fones dos apresentadores vão conectar fisicamente)
  • Input: definir como Zoom H6 — essa é a entrada dos microfones para o OBS

IMPORTANTE: clicar em Properties do dispositivo H6 → alterar sample rate para 48000 Hz, 24 bits. Isso evita o bug clássico de o H6 "sumir" do Windows por conflito de sample rate.

Parear fones Bluetooth dos apresentadores no Windows

Qui 7/5 Urgente

Apresentadores costumam chegar em cima da hora com fone novo na mão. Tutorial pronto pra cada modelo evita pânico de última hora.

Antes de qualquer coisa:

⚠️ Pegadinha clássica do Windows com fone Bluetooth — Stereo vs Hands-Free

Quando um fone BT pareia, o Windows registra DOIS dispositivos de áudio:

  • AirPods Pro Stereo ou Galaxy Buds Stereo → áudio em alta qualidade, só saída (perfeito pra monitorar a live)
  • Hands-Free AG Audio → áudio em qualidade telefonema (8 kHz mono), com microfone ativo. Som FICA HORRÍVEL.

O Windows escolhe automaticamente Hands-Free quando detecta um app que usa microfone (Zoom, OBS, etc) — e estraga o som.

Como forçar Stereo:

  • Botão direito no ícone de som → Sound settings → More sound settings
  • Aba Playback → clicar com botão direito no Hands-Free → Disable (desabilitar — não precisa remover)
  • Confirmar que o dispositivo ativo é o Stereo
  • Repetir pra cada um dos 2 fones

Outros problemas comuns:

  • Fone aparece mas não conecta: remover de "Devices", desligar/ligar Bluetooth do PC, tentar de novo
  • "Couldn't connect": reiniciar o Bluetooth do Windows: Win+R → services.msc → procurar "Bluetooth Support Service" → Restart
  • Áudio cortando / chiado: distância do PC, ou interferência (Wi-Fi 2.4GHz, micro-ondas, outros BT). Tirar fones próximos do roteador
  • Latência alta (delay): inerente ao BT, ~80-150ms é normal. Se for inaceitável, plano B é fone com fio + splitter P2 (R$15)

Se os DOIS apresentadores tiverem fone BT: parear os dois primeiro AQUI, depois ir pra próxima tarefa configurar o split A1/A2 no VoiceMeeter.

Configurar áudio no OBS

Qui 7/5

Garantir que o H6 seja a entrada de áudio e o mix chegue correto no Zoom.

OBS → Settings → Audio:

  • Sample Rate: 48 kHz (deve coincidir com o H6)
  • Channels: Stereo
  • Desktop Audio: VoiceMeeter Output (captura o retorno do Zoom)
  • Mic/Auxiliary Audio: Zoom H6 (entrada dos microfones)

No mixer do OBS: verificar que o H6 não está clipando (picos vermelhos). Ajustar gain no próprio H6, não no OBS.

Ativar Audio Monitoring no OBS para a faixa do Desktop Audio (engrenagem na faixa → Advanced Audio → Monitor Only) — isso manda o retorno para os fones via VoiceMeeter.

Configurar Zoom: vídeo, áudio, fundo virtual e processamentos

Qui 7/5 Urgente

Selecionar câmera virtual do OBS, microfone certo, desativar processamentos que estragam mic profissional.

📹 VÍDEO — Settings → Video

  • Camera: selecionar OBS Virtual Camera no dropdown
    ↳ Antes disso, no OBS: clicar em Start Virtual Camera no canto inferior direito. Se não aparecer no Zoom, reiniciar Zoom depois de iniciar a Virtual Cam.
  • Original ratio: ON (16:9)
  • HD: ON (Habilitar HD)
  • Mirror my video: OFF — IMPORTANTE. Senão participantes vêem espelhado, texto invertido
  • Touch up my appearance: OFF — degrada qualidade
  • Adjust for low light: OFF — você já tem iluminação cuidada, deixar Zoom intacto

🎤 ÁUDIO — Settings → Audio

Tem 2 caminhos válidos. Use Opção A se as animações dos sócios têm áudio que precisa ir pros participantes. Use Opção B se for só voz.

Opção A (recomendada — mix completo):

  • Microphone: VoiceMeeter Output (recebe H6 + áudio das animações OBS)
  • No OBS: na aba Audio, Monitoring Device = VoiceMeeter Aux Input, e nas faixas de áudio (Desktop Audio + Media Source) ativar Monitor and Output
  • No VoiceMeeter: garantir que o input VoiceMeeter Aux tem botão B1 ativo (envia pro Zoom)

Opção B (simples — só voz):

  • Microphone: Zoom H6 direto (bypass VoiceMeeter)
  • Animações com áudio NÃO chegam ao Zoom — só o que apresentadores falam

Speaker / Saída de áudio:

  • ⚠️ NUNCA selecionar VoiceMeeter como Speaker — cria loop de áudio (eco infinito)
  • Selecionar fone do operador (cabo P2 da placa-mãe) ou caixa de som monitor

Processamentos a DESATIVAR (tudo com mic profissional H6 estraga):

  • Automatically adjust microphone volume: OFF
  • Suppress background noise: Low ou Off
  • High fidelity music mode: ON (se disponível no plano — qualidade superior pra apresentação)
  • Echo cancellation: Auto (deixa)
  • Stereo audio: ON (se Pro+)
  • Original sound for musicians: ativar e usar o toggle no canto superior esquerdo da reunião

🌫️ FUNDO VIRTUAL / BLUR — Settings → Background & Effects

  • Blur (desfoque): mais rápido, sem chroma key, funciona em qualquer fundo. Recomendado se PC ainda tem GPU sobrando.
  • Imagem custom: precisa fundo verde (chroma key) ou contraste forte com o apresentador. Sem chroma, recortes com cabelo ficam ruins.
  • ⚠️ Fundo virtual consome GPU. Com 2 câmeras Sony USB + OBS encoder + Zoom rodando, pode estourar. Se notar lag, desativar fundo virtual e investir em iluminação real do cenário.

OUTROS — antes de começar a live:

  • Settings → Recording → marcar Save to local computer + escolher pasta de muito espaço (HD externo)
  • Settings → Recording → marcar Record a separate audio file for each participant (útil pra editar depois)
  • Hotkeys importantes pra decorar:
    • Alt+A — mute/unmute
    • Alt+V — vídeo on/off
    • Alt+S — start/stop screen share
    • Alt+R — record
  • Designar Co-host antes da live: vai ajudar mutar quem entrar tarde, controlar caixinha de espera, gerenciar Q&A se houver
  • Sala de espera (Waiting Room): ativar — controle de quem entra e quando
  • Senha: habilitar (link com password embutido pra simplificar)

📋 Checklist final 10 min antes:

  • OBS rodando + Virtual Cam ON
  • VoiceMeeter aberto + roteamento OK
  • Zoom: câmera = OBS Virtual, mic = VM Output (ou H6), speaker = fone/monitor (não VM)
  • Fundo virtual decidido (blur ou off)
  • Recording configurado
  • Co-host designado
  • Test com participante de fora confirma vídeo + áudio chegando bons

Testar cabo de rede e qualidade da conexão

Sex 8/5 Urgente

Conexão estável é o que separa "live profissional" de "live travada". Cabo > Wi-Fi, sempre.

Conectar o PC principal via cabo Ethernet direto no roteador. Nada de Wi-Fi se for evitável.

Métricas mínimas pra Zoom 1080p estável:

  • Upload: ≥ 5 Mbps sustentado · ideal 10+ Mbps
  • Ping: < 50ms pro servidor mais próximo
  • Jitter (variação): < 30ms

Como testar (na ordem):

  • Fast.com → mostra upload e download. Clicar em "Show more info" pra ver upload.
  • Speedtest.net → mostra ping e jitter detalhados.
  • Monitorar latência durante a live: abrir cmd e rodar ping 8.8.8.8 -t. Deixar correndo num canto da tela. Se pular pra >100ms ou perder pacote, é hora de reagir.

Como melhorar ping antes da live:

  • Fechar tudo que consome banda: OneDrive, Dropbox, Google Drive sync, Steam, downloads, streaming em outros aparelhos da casa
  • Desligar VPN se estiver ativa
  • Windows: Settings → Network → Wi-Fi → Properties — desligar "Metered connection"
  • Se Wi-Fi for inevitável: usar banda 5GHz, ficar perto do roteador, sem parede no caminho

Plano B se tudo falhar: tetherar do celular em 5G/4G. Testar antes pra saber se a operadora sustenta.

↗ Fast.com — teste rápido (sem ads) ↗ Speedtest.net — ping + jitter ↗ Packet Loss Test

Montar e ajustar iluminação

Sex 8/5

Setup 3 pontos é o padrão profissional — sem isso, qualquer câmera de R$30k parece amadora.

Setup clássico de 3 pontos:

  • Key light (luz principal): frontal-lateral a 45° do apresentador, ligeiramente acima da linha dos olhos. É a luz que define o rosto.
  • Fill light (luz de preenchimento): oposta ao key, com ~50% da intensidade. Suaviza as sombras criadas pelo key.
  • Back light / hair light: atrás do apresentador, ilumina ombros e cabelo. Separa visualmente do fundo, dá profundidade.

Temperatura de cor: 4500–5600K (luz neutra/dia). Todas as luzes na mesma temperatura — não misturar quente (3000K) com fria (6500K), o branco da câmera fica dançando.

Difusor é obrigatório: softbox, guarda-chuva, ou rebatedor branco. Luz dura no rosto cria sombras agressivas e marca olheiras.

Cuidado com óculos: se algum apresentador usa óculos, posicionar key light um pouco mais alto e mais lateral pra evitar reflexo na lente. Testar e ajustar enquanto vê o preview no OBS.

Ajustar pelo OBS, não pelo olho: o que parece "OK" a olho nu pode estar overexposto na câmera (rosto estourado). Confiar no preview.

Fundo: ter pelo menos 1 metro de distância entre o apresentador e a parede de fundo. Evita sombra projetada e dá separação.

Conectar microfones e fazer soundcheck

Sex 8/5 Urgente

Áudio ruim mata uma live mais rápido que vídeo ruim — ninguém aguenta 5 minutos de chiado.

Conectar nos canais do Zoom H6:

  • Apresentador 1 → CH1 (lapela ou direcional)
  • Apresentador 2 → CH2
  • Cabos XLR ou TRS conforme o tipo do mic

Phantom power (+48V): ligar SÓ se o mic for condensador. A maioria dos lapelas profissionais usa pilha própria. Mics dinâmicos e shotguns alimentados por pilha NÃO precisam — phantom em mic dinâmico não estraga, mas é desnecessário.

Soundcheck — protocolo de 60 segundos:

  • Pedir cada apresentador falar 30s em volume normal de apresentação (não cochichando, não gritando)
  • Ajustar gain no próprio H6 até pico ficar em -6 dB, com média em -12 a -18 dB
  • NÃO ajustar gain no OBS — sempre na fonte. Subir volume no OBS amplifica o ruído junto com a voz.

Verificar no mixer do OBS:

  • Picos não passam de -6 dB (zona amarela)
  • Sem ruído de fundo audível (se houver, mover mic, isolar fonte do ruído — ar-condicionado, ventilador, geladeira na parede)
  • Os dois canais balanceados — volume parecido entre os apresentadores

Compressão suave (recomendado): em cada canal de mic no OBS, adicionar Filter → Compressor:

  • Ratio: 3:1
  • Threshold: -18 dB
  • Attack: 10 ms · Release: 100 ms
  • Output gain: +3 dB

Isso evita que picos quando alguém empolgar estourem o áudio.

Plano B: ter um mic backup (USB ou outro lapela) na bolsa. Se um cair, troca em 30s sem parar a live.

Configurar PowerPoint e NDI no PC do apresentador

Sex 8/5

Slides via NDI vão direto pro OBS sem cabo HDMI atravessando o estúdio.

No PC do PowerPoint:

  • Abrir o arquivo .pptx da apresentação
  • DESATIVAR Modo Apresentador (Presenter View): Slide Show → desmarcar "Use Presenter View". Se ficar ativo, o NDI captura a tela com notas em vez do slide cheio.
  • Resolução de saída: 1920×1080 (mesma do OBS) — Display Settings → Resolution
  • Se for monitor secundário: configurar Show on monitor X apontando pra tela que o NDI Scan Converter está capturando

Abrir NDI Scan Converter:

  • Ícone na bandeja do sistema (taskbar, canto direito)
  • Selecionar a tela onde o PPT vai rodar em fullscreen
  • Confirmar que o nome do PC aparece na lista de fontes NDI

No PC principal (OBS):

  • Abrir cena Apresentação PPT → confirmar que NDI Source mostra o slide certo
  • Apertar F4 e ver se a transição funciona
  • Se aparecer "No Source" no NDI, verificar firewall do Windows nos dois PCs (NDI usa portas 5353, 5960-5990)

Teste real — rodar 5 slides aleatórios:

  • Sincronia: delay entre clicar "próximo" e aparecer no OBS deve ser < 500ms
  • Sem flicker, sem cortes, sem tearing
  • Texto pequeno legível (não pixelado)

Plano B se NDI falhar: ter cabo HDMI longo + placa de captura USB (Elgato Cam Link ou similar). Plugar HDMI do PC do PPT na placa, capturar via Video Capture Device no OBS.

Confirmar que os dois fones BT estão recebendo o áudio sem problemas.

  • Com a reunião Zoom aberta, verificar que o áudio chega nos dois fones
  • Se um fone tiver delay muito perceptível: isso é inerente ao Bluetooth. Para live de apresentação (não dublagem) um delay de 80–150ms é tolerável. Se for inaceitável, usar fone com fio + splitter P2 como plano B.
  • Ter um cabo P2 splitter disponível como backup físico — custa R$15 e resolve qualquer problema de BT

Fechar apps e processos desnecessários

Urgente

Liberar CPU/RAM/GPU para OBS + Zoom.

Fechar: Chrome, Spotify, OneDrive, Google Drive Sync, Teams (se não for usar), qualquer antivírus pesado.

  • No OBS: Settings → Output → Encoder → NVENC H.264 (usa a 3070 Ti em vez da CPU — essencial com duas câmeras + Zoom rodando)
  • Bitrate de streaming para Zoom: 4500 kbps é suficiente para 1080p30
  • Verificar no Gerenciador de Tarefas que a GPU está sendo usada pelo OBS (coluna GPU deve mostrar atividade no processo obs64.exe)

Troubleshooting ao vivo

SOS · soluções rápidas

Sintoma

Boca do apresentador mexe e o som chega antes ou depois — sincronia labial errada. Participantes do Zoom vêem isso, você muitas vezes não.

⚡ Fix em 10 segundos
  1. OBS → painel Audio Mixer
  2. Engrenagem na faixa do Mic/Aux → Advanced Audio Properties
  3. Coluna Sync Offset (ms) → digitar valor:
    • Positivo (ex: +100) = atrasa áudio. Use se boca mexe ANTES do som.
    • Negativo (ex: -80) = adianta áudio. Use se som chega ANTES da boca.
  4. Começar com 100ms, ajustar de 50 em 50. Pedir alguém em outra máquina pra te avisar quando bater.

Por que aconteceu

  • Bluetooth dos apresentadores adiciona ~80–150ms (tolerável). Se a sua mídia tem áudio, o ouvido do apresentador pode achar que você atrasou — é o BT dele, não você.
  • OBS encoder atolado: CPU/GPU acima de 90% faz frame drop e drift. Verificar Task Manager.
  • Zoom processando localmente: Echo cancellation + AGC podem dessincronizar. Desativar processamentos do Zoom.
  • Câmera USB com encoder lento (YUY2 vs MJPEG): trocar pra MJPEG se for o caso.

Se nada acima resolver

  1. Stop Virtual Cam → Start novamente (no Zoom, re-selecionar câmera depois)
  2. Reduzir resolução do OBS de 1080p pra 720p — alivia encoder
  3. Em último caso: trocar BT por fone com fio + splitter P2 (R$15) — zera o delay

Sintoma

Participantes não te ouvem. VU meter no OBS não mexe ou Zoom mostra "Mic muted".

🔍 Checklist em ordem (10s cada)
  1. H6 ainda conectado? LED do H6 ligado? Cabo USB no PC?
  2. Mic mudo no H6? Botão MUTE de cada canal está ativo?
  3. Mic mudo no apresentador? Lapela tem botão de mute físico
  4. VU meter no OBS: faixa do mic mexe quando alguém fala? Se não, problema é do H6 → Windows
  5. Windows Sound Settings: H6 ainda é o Input default? Sample rate ainda é 48kHz?
  6. VoiceMeeter: volume do Hardware Input 1 não está em -∞?
  7. Zoom muted? Alt+A pra unmute. Verificar canto inferior esquerdo do Zoom.

Restart rápido sem perder a live

  • H6: tirar e recolocar o cabo USB. Windows reidentifica em 3s. OBS reconecta automático.
  • VoiceMeeter travou: Menu → System Settings → Restart Audio Engine (ou Ctrl+R)
  • Bluetooth do Windows: Win+R → services.msc → "Bluetooth Support Service" → Restart
Plano B: tem um mic USB de backup? Conecta direto, troca o Mic Source no OBS e segue. Operadores devem ter SEMPRE 1 mic USB sobressalente em cima da mesa.

Sintoma

Imagem do apresentador congelada / preview do OBS preto / Zoom mostra "Camera not available".

⚡ Fix em ordem
  1. Câmera Sony travou: tirar e recolocar o cabo USB. Se persistir, desligar/ligar o botão da câmera (perde 5s de live, mas resolve)
  2. Virtual Cam congelada: botão direito no preview do OBS → Refresh. Ou: Stop Virtual Camera → Start novamente. No Zoom, re-selecionar OBS Virtual Camera.
  3. OBS preview preto mas câmera funciona: abrir/fechar a Source → ou clicar com botão direito → Properties → Apply (força reload do dispositivo)
  4. OBS travou inteiro: matar processo no Task Manager → reabrir. Scene Collection auto-saved a cada 5min, perde no máximo 5min de configuração.

Causas comuns

  • USB hub interno do gabinete sobrecarregado — câmeras devem estar em portas USB diretas, não em hub
  • Câmera Sony entrou em standby por inatividade — desativar Auto Power Off no menu da câmera
  • Cabo USB com mau contato — testar outro cabo (vale ter sobressalente)
  • Driver do Sony desatualizado no Windows — geralmente OBS 30+ resolve, mas se persistir, atualizar Sony Imaging Edge

Sintoma

Áudio chega abafado, com chiado, cortando, eco, distorcendo, ou volume oscila violentamente.

🎤 Diagnóstico em 30s
  1. Picos vermelhos no mixer OBS? Está clipando — abaixar o gain no H6 (físico), NUNCA no OBS
  2. VU médio em -∞ ou muito baixo? Subir o gain no H6 até média ficar em -12 dB
  3. Áudio cortando intermitente? Se for fone BT, é interferência ou distância do PC
  4. Eco? Outro mic captou a saída do speaker — usar fones em vez de caixa de som
  5. Voz "metálica"? Compressor com ratio alto demais — verificar Filters no OBS, ratio máximo 4:1

Lapela com problema

  • Bateria fraca: ter pilhas/cabos USB-C sobressalentes na mesa
  • Cabo do lapela frouxo: reposicionar — TRS vs TRRS pode invertercanais
  • Vento/ar-condicionado: usar a espumninha (windscreen) — ou redirecionar AC
  • Roupa atritando: reposicionar mic mais alto, longe de tecido
Quando trocar de mic ao vivo: avisar o apresentador, mutar no OBS (ou no H6), trocar fisicamente, voltar audio em 5s. Não tente mudar source no OBS no meio da live — usa o mic backup já configurado em CH3 ou CH4 do H6.

Sintoma

Sua conexão com o Zoom caiu. Apresentadores ainda estão no estúdio mas o Zoom mostra "Reconnecting" ou fechou de vez.

⚡ O que fazer NOS PRÓXIMOS 30 SEGUNDOS
  1. Zoom tenta reconectar sozinho por ~30s. Espera. Não fecha.
  2. Se reconectar: voltar ao normal. Os participantes só viram um freeze de 5–10s.
  3. Se não reconectar: fechar Zoom completamente e abrir o link da live de novo (mesmo link, mesmo PC)
  4. O Co-host designado readmite você automaticamente (foi configurado antes da live)
  5. Reselecionar OBS Virtual Camera + VoiceMeeter Output (Zoom esquece config às vezes)

Plano B agressivo

  • Ter o link do Zoom em mais de um aparelho: PC principal, celular, notebook backup
  • Se PC principal cair total: entrar pelo celular como participante "Operador 2" — pelo menos os apresentadores têm continuidade
  • Apresentadores em outra sala? Eles podem entrar pelo celular tipo "host" enquanto você reativa o estúdio

Internet caiu

  • Tetherar do celular: Settings → Personal Hotspot → 5G/4G → conectar PC via USB ou Wi-Fi
  • Speedtest rápido: fast.com
  • Se foi queda do roteador: alguém vai lá, desliga 10s, liga. Volta em 30-60s.

Checklist 60 segundos antes do "ao vivo"

  1. OBS: Virtual Cam ON, scene ativa correta, mixer mostra VU mexendo
  2. VoiceMeeter: aberto, A1+A2+B1 ativos no input do H6
  3. Zoom:
    • Camera = OBS Virtual Camera
    • Mic = VoiceMeeter Output (ou H6)
    • Speaker NÃO é VoiceMeeter
  4. BTs dos apresentadores: verde, conectado, em modo Stereo (não Hands-Free)
  5. Recording ativo: Alt+R no Zoom + Start Recording no OBS
  6. Co-host designado e online
  7. Lista impressa de sócios com hotkey e nome — em cima da mesa
  8. Janela do ping rodando (ping 8.8.8.8 -t) num canto da tela
  9. Apps fechados: Chrome, Spotify, OneDrive, Drive sync
  10. Celular em silencioso (não vibração — capta no mic)
🎯 Mantra dos 5 segundos

Antes de clicar "ir ao vivo": respira fundo, olha o VU, olha o preview, olha o ping. Se os 4 estiverem OK, vai.
