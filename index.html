Instalar VoiceMeeter Banana

Gratuito

Mixer de áudio virtual que permite rotear o mesmo som para dois fones Bluetooth simultaneamente.

VoiceMeeter Banana é um aplicativo de mixagem de áudio virtual gratuito (donationware — funcional 100%, sem limitações, sem marca d'água). Ele cria dispositivos de áudio virtuais no Windows que você pode rotear para múltiplas saídas físicas.

No nosso caso: A1 = Fone BT 1, A2 = Fone BT 2. Qualquer áudio que entrar pelo Virtual Input será enviado simultaneamente para os dois fones. Essencial para o monitoramento dos apresentadores sem compartilhamento de fones.

Instalar NDI Tools

Gratuito

Permite que o computador do PowerPoint transmita a tela via rede local para o OBS, sem cabos.

Instalar em DOIS computadores: o PC principal (que roda o OBS) e o PC do PowerPoint.

  • No PC do PPT: abrir NDI Scan Converter após instalação — ele começa a transmitir a tela automaticamente via rede local.
  • No PC principal: o plugin OBS-NDI (próxima tarefa) vai detectar essa fonte.
  • Ambos os computadores precisam estar na MESMA rede Wi-Fi ou cabo. Idealmente cabo para estabilidade.
↗ Download NDI Tools — versão gratuita

Instalar plugin OBS-NDI

Gratuito

Plugin que adiciona suporte a fontes NDI dentro do OBS Studio.

Baixar a versão mais recente compatível com seu OBS. Após instalar, reiniciar o OBS.

No OBS, ao adicionar uma nova Source em qualquer cena, vai aparecer a opção NDI Source. Selecionar a fonte que corresponde ao PC do PowerPoint (vai aparecer com o nome do computador).

↗ Releases do plugin OBS-NDI no GitHub

Desativar Windows Update até depois da live

Urgente

Uma atualização automática pode reiniciar o PC no meio da transmissão.

Settings → Windows Update → Advanced Options → Pause updates → selecionar 1 semana. Reativar normalmente após a live.

Também desativar:

Configurar Sony FX30 para modo USB Streaming

Qui 7/5

Ativar o modo que faz a câmera aparecer como webcam no Windows.

  • No menu da FX30: Network → USB → USB Connection Mode → PC Remote
  • Depois: Network → PC Remote Settings → Still Img Save Dest → Memory Card Only (para não travar)
  • Conectar via cabo USB-C → USB-A ou USB-C → USB-C diretamente na porta TRASEIRA do gabinete (não hub, não frente)
  • A câmera vai aparecer no Windows como Sony Camera no Gerenciador de Dispositivos
  • Resolução recomendada: 1080p / 30fps no streaming USB (suficiente para a live, menos carga no barramento USB)
↗ Manual oficial Sony FX30 — USB Streaming

Configurar Sony A7S III para modo USB Streaming

Qui 7/5

Mesmo processo da FX30, mas o menu é levemente diferente.

  • Menu → Setup (maleta) → USB → USB Connection Mode → PC Remote
  • Conectar em outra porta USB do gabinete — idealmente em controlador USB diferente do FX30 (portas frontais vs traseiras geralmente usam controladores diferentes num i9)
  • Verificar no Gerenciador de Dispositivos que AMBAS aparecem como dispositivos de câmera separados
  • Se as duas aparecerem com o mesmo nome no OBS, diferenciar pelo índice: Device 0 = primeira conectada, Device 1 = segunda
↗ Manual oficial Sony A7S III — USB Connection

Testar as duas câmeras simultaneamente no OBS

Qui 7/5

Confirmar que não há conflito de USB antes do dia da live.

  • No OBS: criar uma cena de teste, adicionar dois Video Capture Device

  • Checar no Gerenciador de Tarefas (aba Performance → GPU) se o preview está usando a GPU (deve estar, com 3070 Ti)
  • Configurar resolução/FPS no OBS para cada fonte: 1920x1080, 30fps, formato MJPEG (mais eficiente que YUY2 em USB)

Configurar Zoom H6 como interface de áudio padrão

Qui 7/5

Definir o H6 como entrada e saída de áudio do Windows.

  • Conectar o H6 via USB. Ele aparece como Zoom H6 no Windows — sem driver adicional necessário.
  • Botão direito no ícone de som da barra de tarefas → Sound Settings
  • Output: pode ficar na saída da placa-mãe OU no H6 (depende de onde os fones dos apresentadores vão conectar fisicamente)
  • Input: definir como Zoom H6 — essa é a entrada dos microfones para o OBS

IMPORTANTE: clicar em Properties do dispositivo H6 → alterar sample rate para 48000 Hz, 24 bits. Isso evita o bug clássico de o H6 "sumir" do Windows por conflito de sample rate.

Configurar VoiceMeeter Banana para dois Bluetooth

Qui 7/5

Rotear o áudio do Zoom para os dois fones BT simultaneamente.

  • Parear os dois fones BT no Windows antes de abrir o VoiceMeeter (Settings → Bluetooth)
  • Abrir VoiceMeeter Banana. No canto superior direito:
    • Hardware Out A1 → selecionar Fone BT 1
    • Hardware Out A2 → selecionar Fone BT 2
  • Na faixa Virtual Input (centro): clicar nos botões A1 e A2 para ativar roteamento para ambos
  • No Windows: Sound Settings → Output → selecionar VoiceMeeter Input como padrão
  • No OBS: Audio Settings → Monitoring Device → VoiceMeeter Aux Input (assim você também consegue monitorar o mix do OBS)
↗ Manual completo VoiceMeeter Banana (PDF)
↗ Abrir no YouTube se não carregar aqui

Configurar áudio no OBS

Qui 7/5

Garantir que o H6 seja a entrada de áudio e o mix chegue correto no Zoom.

OBS → Settings → Audio:

  • Sample Rate: 48 kHz (deve coincidir com o H6)
  • Channels: Stereo
  • Desktop Audio: VoiceMeeter Output (captura o retorno do Zoom)
  • Mic/Auxiliary Audio: Zoom H6 (entrada dos microfones)

No mixer do OBS: verificar que o H6 não está clipando (picos vermelhos). Ajustar gain no próprio H6, não no OBS.

Ativar Audio Monitoring no OBS para a faixa do Desktop Audio (engrenagem na faixa → Advanced Audio → Monitor Only) — isso manda o retorno para os fones via VoiceMeeter.

0/0

Criar Scene Collection "Live Sócios"

Qui 7/5

Isolar essa configuração para não bagunçar outras cenas já existentes no OBS.

OBS → Scene Collection → New → nomear Live Sócios 2025. Isso cria um ambiente limpo, separado de qualquer outra configuração existente.

Criar cenas base de câmera

Qui 7/5

As cenas fixas de câmera que serão usadas durante a apresentação.

Criar as seguintes cenas (botão + no painel de Scenes):

  • CAM1 - Principal → Video Capture Device (FX30) em full frame
  • CAM2 - Close → Video Capture Device (A7S3) em full frame
  • CAM1 + CAM2 → FX30 grande + A7S3 em PiP (canto inferior direito, 320x180)
  • Apresentação PPT → NDI Source apontando para o PC do PowerPoint
  • Aguarde → Image Source com tela de espera (texto + logo)

Hotkeys: Settings → Hotkeys → para cada cena, definir tecla de atalho (ex: F1–F5).

Criar as 20 cenas de animação de sócios

Qui 7/5

Uma cena por sócio, cada uma com o arquivo de vídeo correspondente.

  • Organizar arquivos: criar pasta C:\LiveSocios\Animacoes\ e nomear: socio_01.mp4 ... socio_20.mp4
  • Para cada sócio: criar cena SÓCIO 01 — [Nome], adicionar Media Source → apontar para o arquivo correto
  • Em cada Media Source: marcar "Restart playback when source becomes active" e DESMARCAR "Loop"
  • Hotkeys: Num1–Num0 para sócios 1–10, Shift+Num1–Num0 para sócios 11–20

Dica: adicionar também uma fonte de Audio em cada cena de animação se os vídeos tiverem áudio — verificar volume no mixer.

Configurar transições

Qui 7/5

Transições suaves entre câmeras e animações.

OBS → Scene Transitions (parte de baixo do painel de cenas):

  • Câmera → Câmera: Fade, 200ms
  • Câmera → Animação: Cut (0ms) — o vídeo começa imediatamente, sem fade que pareceria estranho
  • Animação → Câmera: Fade, 300ms — sensação de encerramento da animação

Testar cabo de rede e qualidade da conexão

Sex 8/5 Urgente

Conexão estável é o que separa "live profissional" de "live travada". Cabo > Wi-Fi, sempre.

Conectar o PC principal via cabo Ethernet direto no roteador. Nada de Wi-Fi se for evitável.

Métricas mínimas pra Zoom 1080p estável:

  • Upload: ≥ 5 Mbps sustentado · ideal 10+ Mbps
  • Ping: < 50ms pro servidor mais próximo
  • Jitter (variação): < 30ms

Como testar (na ordem):

  • Fast.com → mostra upload e download. Clicar em "Show more info" pra ver upload.
  • Speedtest.net → mostra ping e jitter detalhados.
  • Monitorar latência durante a live: abrir cmd e rodar ping 8.8.8.8 -t. Deixar correndo num canto da tela. Se pular pra >100ms ou perder pacote, é hora de reagir.

Como melhorar ping antes da live:

  • Fechar tudo que consome banda: OneDrive, Dropbox, Google Drive sync, Steam, downloads, streaming em outros aparelhos da casa
  • Desligar VPN se estiver ativa
  • Windows: Settings → Network → Wi-Fi → Properties — desligar "Metered connection"
  • Se Wi-Fi for inevitável: usar banda 5GHz, ficar perto do roteador, sem parede no caminho

Plano B se tudo falhar: tetherar do celular em 5G/4G. Testar antes pra saber se a operadora sustenta.

↗ Fast.com — teste rápido (sem ads) ↗ Speedtest.net — ping + jitter ↗ Packet Loss Test

Montar e ajustar iluminação

Sex 8/5

Setup 3 pontos é o padrão profissional — sem isso, qualquer câmera de R$30k parece amadora.

Setup clássico de 3 pontos:

  • Key light (luz principal): frontal-lateral a 45° do apresentador, ligeiramente acima da linha dos olhos. É a luz que define o rosto.
  • Fill light (luz de preenchimento): oposta ao key, com ~50% da intensidade. Suaviza as sombras criadas pelo key.
  • Back light / hair light: atrás do apresentador, ilumina ombros e cabelo. Separa visualmente do fundo, dá profundidade.

Temperatura de cor: 4500–5600K (luz neutra/dia). Todas as luzes na mesma temperatura — não misturar quente (3000K) com fria (6500K), o branco da câmera fica dançando.

Difusor é obrigatório: softbox, guarda-chuva, ou rebatedor branco. Luz dura no rosto cria sombras agressivas e marca olheiras.

Cuidado com óculos: se algum apresentador usa óculos, posicionar key light um pouco mais alto e mais lateral pra evitar reflexo na lente. Testar e ajustar enquanto vê o preview no OBS.

Ajustar pelo OBS, não pelo olho: o que parece "OK" a olho nu pode estar overexposto na câmera (rosto estourado). Confiar no preview.

Fundo: ter pelo menos 1 metro de distância entre o apresentador e a parede de fundo. Evita sombra projetada e dá separação.

Posicionar câmeras e verificar enquadramento

Sex 8/5

FX30 no plano aberto, A7S3 no close — ajustar foco antes de começar.

  • Iluminação: a câmera vai compensar automaticamente (Auto ISO), mas se possível adicionar luz frontal difusa
  • FX30: Auto Focus com Eye Tracking se disponível — mantém apresentadores focados
  • A7S3: foco contínuo ativado
  • No OBS: verificar preview da cena CAM1 + CAM2 para confirmar enquadramento e exposição

Conectar microfones e fazer soundcheck

Sex 8/5 Urgente

Áudio ruim mata uma live mais rápido que vídeo ruim — ninguém aguenta 5 minutos de chiado.

Conectar nos canais do Zoom H6:

  • Apresentador 1 → CH1 (lapela ou direcional)
  • Apresentador 2 → CH2
  • Cabos XLR ou TRS conforme o tipo do mic

Phantom power (+48V): ligar SÓ se o mic for condensador. A maioria dos lapelas profissionais usa pilha própria. Mics dinâmicos e shotguns alimentados por pilha NÃO precisam — phantom em mic dinâmico não estraga, mas é desnecessário.

Soundcheck — protocolo de 60 segundos:

  • Pedir cada apresentador falar 30s em volume normal de apresentação (não cochichando, não gritando)
  • Ajustar gain no próprio H6 até pico ficar em -6 dB, com média em -12 a -18 dB
  • NÃO ajustar gain no OBS — sempre na fonte. Subir volume no OBS amplifica o ruído junto com a voz.

Verificar no mixer do OBS:

  • Picos não passam de -6 dB (zona amarela)
  • Sem ruído de fundo audível (se houver, mover mic, isolar fonte do ruído — ar-condicionado, ventilador, geladeira na parede)
  • Os dois canais balanceados — volume parecido entre os apresentadores

Compressão suave (recomendado): em cada canal de mic no OBS, adicionar Filter → Compressor:

  • Ratio: 3:1
  • Threshold: -18 dB
  • Attack: 10 ms · Release: 100 ms
  • Output gain: +3 dB

Isso evita que picos quando alguém empolgar estourem o áudio.

Plano B: ter um mic backup (USB ou outro lapela) na bolsa. Se um cair, troca em 30s sem parar a live.

Configurar PowerPoint e NDI no PC do apresentador

Sex 8/5

Slides via NDI vão direto pro OBS sem cabo HDMI atravessando o estúdio.

No PC do PowerPoint:

  • Abrir o arquivo .pptx da apresentação
  • DESATIVAR Modo Apresentador (Presenter View): Slide Show → desmarcar "Use Presenter View". Se ficar ativo, o NDI captura a tela com notas em vez do slide cheio.
  • Resolução de saída: 1920×1080 (mesma do OBS) — Display Settings → Resolution
  • Se for monitor secundário: configurar Show on monitor X apontando pra tela que o NDI Scan Converter está capturando

Abrir NDI Scan Converter:

  • Ícone na bandeja do sistema (taskbar, canto direito)
  • Selecionar a tela onde o PPT vai rodar em fullscreen
  • Confirmar que o nome do PC aparece na lista de fontes NDI

No PC principal (OBS):

  • Abrir cena Apresentação PPT → confirmar que NDI Source mostra o slide certo
  • Apertar F4 e ver se a transição funciona
  • Se aparecer "No Source" no NDI, verificar firewall do Windows nos dois PCs (NDI usa portas 5353, 5960-5990)

Teste real — rodar 5 slides aleatórios:

  • Sincronia: delay entre clicar "próximo" e aparecer no OBS deve ser < 500ms
  • Sem flicker, sem cortes, sem tearing
  • Texto pequeno legível (não pixelado)

Plano B se NDI falhar: ter cabo HDMI longo + placa de captura USB (Elgato Cam Link ou similar). Plugar HDMI do PC do PPT na placa, capturar via Video Capture Device no OBS.

Ensaio completo com o Zoom

Sex 8/5

Entrar na reunião Zoom de teste e validar vídeo + áudio chegando corretos.

  • No Zoom: Settings → Video → selecionar OBS Virtual Camera (ativar no OBS: Start Virtual Camera)
  • Settings → Audio → Microphone → VoiceMeeter Output (ou H6 diretamente)

Esses processamentos do Zoom degradam a qualidade do H6. Pedir para alguém entrar na reunião de teste e confirmar vídeo/áudio.

Verificar monitoramento dos apresentadores

Sex 8/5 Risco

Confirmar que os dois fones BT estão recebendo o áudio sem problemas.

  • Com a reunião Zoom aberta, verificar que o áudio chega nos dois fones
  • Se um fone tiver delay muito perceptível: isso é inerente ao Bluetooth. Para live de apresentação (não dublagem) um delay de 80–150ms é tolerável. Se for inaceitável, usar fone com fio + splitter P2 como plano B.
  • Ter um cabo P2 splitter disponível como backup físico — custa R$15 e resolve qualquer problema de BT

Fechar apps e processos desnecessários

Urgente

Liberar CPU/RAM/GPU para OBS + Zoom.

Fechar: Chrome, Spotify, OneDrive, Google Drive Sync, Teams (se não for usar), qualquer antivírus pesado.

  • No OBS: Settings → Output → Encoder → NVENC H.264 (usa a 3070 Ti em vez da CPU — essencial com duas câmeras + Zoom rodando)
